## Як працює інтеграція

BaseLinker є центральним джерелом даних для магазину FixTar:

```
BaseLinker (склад, ціни, товари)
    |
    |-- sync --> Turso DB (локальна копія) --> Магазин FixTar (фронтенд)
    |
    |<-- push -- Замовлення з магазину
```

- **Товари** синхронізуються з BaseLinker до локальної бази даних (Turso)
- **Замовлення**, оформлені в магазині, передаються до BaseLinker
- **Статуси замовлень** завантажуються з BaseLinker і відображаються клієнту
- **Ціни та залишки на складі** — єдине джерело правди — завжди BaseLinker

## Товари

### Синхронізація

Товари завантажуються з обраного inventory в BaseLinker і зберігаються в базі Turso.

**Що синхронізується:**
- Назва, опис (HTML видаляється автоматично)
- Ціна (з групи цін за замовчуванням)
- Зображення (перше зображення зі списку)
- Категорія
- Залишки на складі (сума з усіх складів)
- SKU, EAN, вага
- Теги

**Як запустити sync:**
1. Перейдіть на `/admin/baselinker`
2. Оберіть inventory зі списку
3. Натисніть "Dry Run", щоб побачити що зміниться (без запису)
4. Натисніть "Synchronizuj", щоб зберегти товари в базу

**Або через API:**
```
POST /api/baselinker/sync
Body: { "inventoryId": 12345, "dryRun": false }
```

### Додавання нових товарів

1. Додайте товар у BaseLinker (Produkty > Magazyn)
2. Заповніть: назву, опис, ціну, зображення, категорію, залишки
3. Запустіть синхронізацію в панелі адміна
4. Товар з'явиться в магазині автоматично

### Редагування товарів

Усі зміни робимо **в BaseLinker**, не в магазині:
- Зміна ціни → BaseLinker > Produkty > редагувати ціну
- Зміна опису → BaseLinker > Produkty > редагувати опис
- Зміна зображення → BaseLinker > Produkty > редагувати зображення
- Після змін: запустити sync

### Видалення товарів

Видаліть товар з BaseLinker і запустіть sync — товар буде оновлений (залишок = 0).

## Акції та знижки

Акції повністю керуються через BaseLinker. Магазин автоматично показує товари зі зниженою ціною на сторінці `/deals`.

### Як додати акцію

1. У BaseLinker відкрийте редагування товару
2. Встановіть **звичайну ціну** (вищу) в полі `original_price` / початкова ціна
3. Встановіть **акційну ціну** (нижчу) в полі ціни
4. Запустіть синхронізацію

**Результат у магазині:**
- Товар з'явиться на сторінці `/deals`
- Відобразиться закреслена оригінальна ціна + акційна ціна
- Автоматичний бейдж з відсотком знижки (напр. "-25%")
- На головній сторінці з'явиться банер "Promocje — X produktów w niższej cenie"

### Як завершити акцію

1. У BaseLinker видаліть початкову ціну (original_price) або встановіть її рівною ціні продажу
2. Запустіть sync
3. Товар зникне зі сторінки `/deals`

### Логіка відображення

- Сторінка `/deals` показує ТІЛЬКИ товари, де `original_price > price`
- Сортування: найвищий відсоток знижки зверху
- Банер на головній: видимий коли є >= 1 товар в акції
- Коли акцій немає: банер змінюється на "Darmowa dostawa od 299 zł"

## Замовлення

### Потік замовлення

```
1. Клієнт оформляє замовлення в магазині
2. Замовлення зберігається в базі FixTar (статус: pending)
3. Клієнт оплачує (P24 redirect або накладений платіж)
4. Webhook P24 підтверджує оплату → статус: processing
5. Замовлення передається до BaseLinker (pushOrder)
6. Обробка в BaseLinker (пакування, відправка)
7. Статуси синхронізуються назад у магазин
```

### Статуси замовлень

Магазин мапить статуси BaseLinker на внутрішні:

| Статус у магазині | Ключові слова BaseLinker |
|---|---|
| `delivered` | dostarczono, odebrano |
| `shipped` | wysłano, nadano |
| `processing` | w realizacji, opłacono, kompletowanie |
| `cancelled` | anulowano, zwrot |
| `pending` | nowe, oczekujące |

Маппінг автоматичний на основі польських назв статусів у BaseLinker.

### Замовлення з Allegro / Temu

Замовлення з Allegro і Temu потрапляють до BaseLinker автоматично. Коли клієнт магазину FixTar увійде з тим самим email, він також побачить ці замовлення у вкладці "Moje zamówienia" (завантажуються з BaseLinker за email-адресою).

### pushOrder — режим розробки

Зараз `pushOrder` працює в режимі **mock** — не створює справжніх замовлень у BaseLinker. Це навмисно, оскільки BaseLinker підключений до живого Allegro.

Щоб увімкнути: відредагуйте `src/lib/services/baselinker.ts`, метод `pushOrder()` — розкоментуйте справжній виклик API.

## Методи доставки

Методи доставки зберігаються в базі даних (таблиця `shipping_methods`), не в BaseLinker.

### Поточні методи

| Метод | Ціна | Безкоштовно від | Час | Назва в BL |
|---|---|---|---|---|
| InPost Paczkomat | 14,99 zł | 299 zł | ~2 дні | Inpost |
| Kurier DPD | 16,99 zł | 299 zł | ~2 дні | DPD |
| ORLEN Paczka | 10,99 zł | 299 zł | ~3 дні | Allegro Automat ORLEN Paczka |

### Зміна цін / додавання методів

Редагуйте безпосередньо в базі Turso:

```sql
-- Зміна ціни (в грошах!)
UPDATE shipping_methods SET cost = 1299 WHERE id = 'inpost-paczkomat';

-- Зміна порогу безкоштовної доставки
UPDATE shipping_methods SET free_shipping_threshold = 19900 WHERE id = 'dpd-kurier';

-- Додавання нового методу
INSERT INTO shipping_methods (id, name, description, cost, free_shipping_threshold, estimated_days, baselinker_name, enabled, sort_order)
VALUES ('dhl-kurier', 'Kurier DHL', 'Dostawa DHL', 1899, 29900, 2, 'DHL', 1, 4);

-- Вимкнення методу (не видаляйте — вимкніть)
UPDATE shipping_methods SET enabled = 0 WHERE id = 'orlen-paczka';
```

**Важливо:** Ціни та пороги вказуються в грошах (1499 = 14,99 zł).

Колонка `baselinker_name` відповідає полю `delivery_method` у замовленні BaseLinker.

## Оплата (Przelewy24)

### Поточний стан

- **Оплата при отриманні (COD)** — працює, комісія +5 zł
- **Przelewy24 (BLIK, картка, переказ)** — готово, чекає на облікові дані

### Налаштування P24

1. Увійдіть до панелі Przelewy24: https://panel.przelewy24.pl
2. Отримайте дані:
   - Merchant ID (ID продавця)
   - POS ID (зазвичай такий самий як Merchant ID)
   - Ключ CRC
   - Ключ API
3. Додайте до файлу `.env`:

```env
P24_MERCHANT_ID=123456
P24_POS_ID=123456
P24_CRC=abcdef1234567890
P24_API_KEY=abcdef1234567890abcdef
```

4. Перезапустіть сервер
5. У checkout автоматично з'являться BLIK, картка та онлайн-переказ

### Webhook P24

Przelewy24 має бути налаштовано для надсилання webhook на:

```
https://ваш-домен.pl/api/webhooks/payment?provider=przelewy24
```

Встановіть цю URL у панелі P24 як "URL statusu transakcji".

### Потік оплати

```
1. Клієнт обирає BLIK/картку/переказ у checkout
2. Магазин реєструє транзакцію в P24 API
3. P24 повертає URL → клієнт перенаправляється
4. Клієнт оплачує на сторінці P24
5. P24 надсилає webhook до магазину → замовлення підтверджено
6. Клієнт повертається на /checkout/success
```

### Тестування

Без даних P24 у `.env` єдиний доступний метод — "Оплата при отриманні". Після додавання даних онлайн-методи з'являються автоматично — код змінювати не потрібно.

## Панель адміністратора

### /admin/baselinker

- Перевірка статусу з'єднання з BaseLinker
- Вибір inventory для синхронізації
- Dry run (перегляд змін без запису)
- Повна синхронізація товарів

### Доступ до бази

База Turso доступна через:
- Turso CLI: `turso db shell fixtar`
- Dashboard: https://turso.tech/app

## FAQ

**П: Як часто синхронізувати товари?**
В: Після кожної зміни в BaseLinker (ціни, описи, нові товари). Також можна налаштувати cron кожні кілька годин.

**П: Чи перезапише редагування товару в магазині дані?**
В: Редагування товарів у магазині неможливе. Єдине джерело — BaseLinker. Наступний sync перезапише будь-які ручні зміни в базі.

**П: Що якщо клієнт замовить товар, якого вже немає?**
В: Залишки на складі актуальні на момент останнього sync. Для критичних товарів синхронізуйте частіше.

**П: Чи видно замовлення з Allegro в магазині?**
В: Так — якщо клієнт використає ту саму email-адресу. Замовлення завантажуються з BaseLinker і відображаються в "Moje zamówienia".

**П: Як змінити ціну доставки?**
В: Відредагуйте таблицю `shipping_methods` у базі Turso. Ціни вказуються в грошах.

**П: Як додати новий метод оплати?**
В: Зараз підтримуються: Przelewy24 (BLIK, картка, переказ) та COD. Додавання PayU або іншого провайдера вимагає реалізації нового `PaymentProvider` у `src/lib/services/payment/`.
